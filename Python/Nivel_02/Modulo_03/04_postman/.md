    - fazer o login é algo bem interessante para deixar salvo as coleções criadas

# O que são coleções?
    são um grupamento de requisições

# Como testar as requisições no Postman?
    usando a url da api | no swagger > file > save as yaml
    no Postman: import > fazer login >  click drop anywhere to import ou arrastar o arquivo > postman collection > import

# Como criar uma nova terefa?
    Na seta no canto superior direito tem uma setinha, e ela selecionamos "No Environment" (ambiente que estamos fazendo)

# Como criar um Environments?
    Click Environments > + > nome environments > 

#    Variable    |   Type    |   Initial value   |   Current value
-    baseUrl     |   def...  |   valor INITIAL do baseUrl: (http://124.0.0.1:5000)    |   mesmo valor: (http://124.0.0.1:5000)

#   atalho para salvar (ctrl + s), depois de salvar a bolinha laranja vai sumir e já pode fechar
    agora é possível selecionar o Taks local na setinha do lado superior direito, 
    e caso click no olho do lado "Environment quick look", é possível visualizar suas variáveis baseUrl, também é possível fazer a alteração do valor

#   Cada linha é uma requisição (GET/POST/PUT/PATCH/DELETE/HEAD/OPTIONS)
    mas na api desse método não é necessário alterar, também tem o corpo (body)

#   Oque preciso entender?
    é que com o Postman, da pra fazer requisições para testa-las
    retorna:
        código da resposta
        delay
        tamanho da resposta
        e tudo referente a essa resposta: Body | Cookies | Headers | Test Results
    um softwer para ajudar a testes, recomendado a usar em vez do swagger, para qualquer api ou de uma api que queira utilizar
    também é possível usar sem ser local, só usar o url da produção, só criar um novo environments
